FAQs

How do alerts work today?

Current public pages present the production portal and roadmap. Interactive module actions are intentionally marked as coming soon while the safety workflows continue through staged implementation.

Why are the module buttons disabled?

The buttons show planned capability placement without implying that the live feature is already available. This avoids misleading users and keeps the portal review-friendly.

What permissions should users expect?

Location access should only be requested when needed for a specific safety function. The intended model is minimal, contextual access rather than continuous background collection.

Troubleshooting checklist

  • Refresh the page to ensure the latest static build is loaded from GitHub Pages.
  • Confirm that your browser is not blocking styles and that ./styles.css loads successfully.
  • Check that you are opening the root production URLs, not an older subfolder path.
  • If a module is marked coming soon, treat that as expected current behavior.
  • For urgent incidents, rely on official local emergency direction first.

Permissions guidance

Kahu Ola is designed to request the least access necessary for a specific feature. If location-related functionality is introduced, users should grant permission only when the feature is actively needed and should review device settings if behavior appears inconsistent.

Privacy reminder

Location data is processed via Edge Computing on the user’s device (on-device) or an equivalent edge path; we do not store raw precise location on our servers by default.

This privacy-first model is intended to support safety use cases while reducing centralized exposure of sensitive location information.
