Edge Processing & Privacy summary

Location data is processed via Edge Computing on the user’s device (on-device) or an equivalent edge path; we do not store raw precise location on our servers by default.

This means Kahu Ola is designed to evaluate location context locally for safety features rather than retaining a default server-side history of precise user movement.

FAQ

Alerts

Kahu Ola is designed to clarify public information, not replace official emergency alerts. If alerts feel delayed or incomplete, always check direct guidance from authorized local agencies.

Modules

The module grid on the landing page shows the planned production surface for sixteen tools. Buttons are intentionally disabled and labeled as coming soon so the portal stays honest about current availability.

Permissions

If you grant location permission in a related app experience, Kahu Ola uses that context for safety relevance only. The default privacy posture is on-device Edge Computing, not background storage of raw precise coordinates on servers.

Troubleshooting checklist

  • Confirm you are loading the production URLs directly from https://kahuola.org/.
  • Refresh the page if the latest layout or styles do not appear immediately after a deploy.
  • Check that CSS is loading from ./styles.css and not from an outdated cached copy.
  • Verify that you are opening the root privacy and support pages, not older nested pages.
  • When reporting an issue, include screenshots and the page URL you visited.
