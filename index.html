Operational Notes

Kahu Ola uses a conservative presentation model on static hosting. The board communicates system intent clearly: public-source integrations are treated as reference signals, and community users are never misled into thinking this page is a live command dashboard.

  • Static pages avoid fragile browser-side network dependencies.
  • Clear status labeling supports review by partners and auditors.
  • Operational decisions should still follow official Maui emergency guidance.

About the Project

Built for Maui County, operated as independent civic technology

Kahu Ola is a community-centered product vision for safety readiness in Maui. It is designed to complement official alerts with a calmer interface, clearer decision paths, and practical modules that reduce friction during high-stress moments.

The project remains independent. It references public data sources such as NASA, NOAA, and USGS, but it is not owned, operated, or endorsed by those agencies. The goal is to translate public information into a format that is easier for local residents and families to use.

Mission

Deliver a high-trust civic interface that improves situational understanding for fire, weather, air quality, movement constraints, and recovery planning across Maui County.

  • Independent civic-tech positioning with clear public-source attribution.
  • Kupuna-friendly design with strong contrast and readable structure.
  • Support contact: long@kahuola.org.

Privacy Support

Privacy Summary

Location data is processed via edge methods designed to keep precise location handling on the user side by default. Kahu Ola is built to avoid unnecessary retention of raw precise location data on centralized servers.

  • Clear audit language for privacy reviewers.
  • Direct access to support and policy pages from the root domain.
  • Static deployment model suited to GitHub Pages root hosting.
